## Bugs (reproducible)

### B1 · Billing invoices list broken (HIGH)
- Where: `/provider/billing`, `src/pages/provider/Billing.tsx:57`.
- Reproduce: navigate as provider → red toast + empty list.
- Cause: PostgREST embed `customers(name,company_name)` requires a foreign key on `invoices.customer_id` referencing `customers.id`. Table has zero FKs.
- Fix options: (a) add the FK in a migration, or (b) drop the embed and hydrate customers with a second query keyed on `customer_id`.

### B2 · Zone `description` never shipped (HIGH — feature/spec mismatch)
- Where: `src/components/provider/ZonesSettings.tsx` and `public.service_zones`.
- Reproduce: Settings → Zones → *New Zone*. Dialog only has Name + color swatches.
- SQL: `service_zones` columns are `id, tenant_id, name, color, created_at, updated_at` — no `description`.
- Impact: TEST_PLAN §3 and §13 explicitly require a description text area (per the earlier plan/AI-message thread). Feature was planned but neither the column nor UI exists.

### B3 · Customer email history returns 403 for PROVIDER_ADMIN (MEDIUM)
- Where: Customer detail card "Emails sent to this customer" (`CustomerEmailHistoryTab.tsx:15`).
- Reproduce: open any customer as `test@acme.io` → RPC `get_customer_email_history` returns 403 in network tab. UI silently shows "No emails yet.".
- Cause: RPC is `SECURITY DEFINER` with `GRANT EXECUTE ... TO authenticated`, but its body likely restricts to super_admin or the customer owner. Provider admins cannot see mail sent from their own tenant to their customers.

### B4 · Sidebar labels not translated (LOW UX, LOW effort)
- Where: `ProviderSidebar.tsx` (nav item labels are hard-coded EN strings). i18n keys exist under `src/i18n/locales/ro/provider.json` but the sidebar doesn't consume them.
- Impact: with locale=RO the shell looks half-translated (screenshots 01–12 all show it).

### B5 · Customer detail sections aren't tabs (LOW — spec mismatch)
- Where: `src/pages/provider/CustomerDetail.tsx`.
- Observation: Properties / Contracts / Service Visits render as stacked sections, not tabs. TEST_PLAN §11 assumes tabs (`get_by_role('tab', name='Properties')` timed out during automation). Ordering (Properties → Contracts → Visits) matches spec.
- Decide: update the spec to "sections", or reintroduce tabs.

## UX friction — top 5 (ranked)

1. **No property‑creation shortcut from empty customer.** QA Test Client shows "No properties yet" + `Add Property` button, but the button is far from the "Inactive" badge that explains *why* the customer is inactive. Suggest promoting a one‑line hint under the badge: *"Add a property, then create a contract to activate."*
2. **New Contract page hides property picker until customer is chosen** — the empty right‑hand Summary panel dominates the fold. Move the Summary below on <1280px or collapse it until Customer is set.
3. **Zone dialog gives no way to describe/annotate a zone** — providers must remember polygon coverage from memory (see B2).
4. **Billing error surfaces only as a toast** that disappears in 4s; the table below just says empty state. Add an inline error banner when the fetch fails so the state is not silently misinterpreted as "no invoices".
5. **Language switcher is buried in the header globe icon with no label**; combined with B4, a first‑run RO provider still sees English nav. Add a first‑login toast: *"Continuă în română? [Salvează]"* if `profiles.locale` was auto‑detected but nav strings are EN.
